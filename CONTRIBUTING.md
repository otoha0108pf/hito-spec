# 参加の手引き / Contributing Guide

## フィードバックの方法 / How to Give Feedback

### GitHub Issue（推奨 / Recommended）

以下のような内容を歓迎します:

- 規格の矛盾や不明瞭な箇所の指摘
- 実装してみた際の疑問や課題
- 関連する研究・規格の情報提供
- 翻訳の改善提案

We welcome:

- Reports of inconsistencies or unclear sections
- Questions or challenges encountered during implementation
- Pointers to related research or specifications
- Suggestions for translation improvements

### プルリクエスト / Pull Requests

規格本文への変更を提案できます。ただし:

- マージには管理者の明示的な承認が必要です
- 日本語が正典です。英語のみのPRは、日本語側への反映方法を含めてください
- 大きな変更は、先にIssueで議論してからPRにしてください

PRs to the specification text are welcome, but:

- Merging requires explicit approval from the maintainer
- Japanese is the canonical language. English-only PRs should include how the change maps to the Japanese text
- For large changes, please open an Issue for discussion before submitting a PR

## 何が変更可能か / What Can Be Changed

| 対象 / Target | プロセス / Process |
|---|---|
| 誤記・表記揺れ / Typos | PR歓迎。即マージ可能 / PR welcome, may merge quickly |
| 明確化・補足 / Clarifications | Issue → 議論 → PR / Issue → discussion → PR |
| 新しい章・概念 / New chapters or concepts | Issue必須。設計セッションで検討 / Issue required, reviewed in design sessions |
| 破壊的変更 / Breaking changes | Issue必須。移行パスの提案を含めること / Issue required, must include migration path |
